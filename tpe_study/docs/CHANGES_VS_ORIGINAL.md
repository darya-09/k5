## 1. ГЛАВНОЕ изменение: исправлен дефект расчёта градиента (fin_4, fin_5)
В оригинале вес кандидата считается так (`fin_4`/`fin_5`, функция `_as_2d_points`):
```python
def _as_2d_points(x_points):
    ...
    if arr.ndim == 1:
        return np.column_stack([arr, np.zeros_like(arr)])   # <-- вторая координата = 0
```
с вашим же комментарием:
> «Если TPE передаёт только одномерный массив, полноценный 2D-градиент восстановить нельзя.
> Для совместимости используется проекция x1=0.0. **Для статьи нужно отдельно проверить, как
> именно ваш TPEOptimizer вызывает weight_fn.**»

Следствие: когда оптимизатор передавал точки покоординатно (1-D), `‖∇f‖` для веса вычислялся
в точке `(x, 0)`, а **не** в реальной точке кандидата. То есть вес w(x) во всех формах опирался на
**неверный градиент**.

**В `tpe_study`:** градиент берётся в РЕАЛЬНОЙ 2-D точке кандидата
(`src/tpe_study/tpe.py`, веса считаются по `grad_fn(x)` для фактического `x`). Поэтому мой вывод
«формы веса не дают значимого улучшения» получен на КОРРЕКТНОМ градиенте, а не на дефектном.

## 2. Одна реализация TPE вместо внешней библиотеки + разрозненных ноутбуков
- Оригинал: внешний `TPEOptimizer` (пакет `tpe`), параметры спрятаны в библиотеке; 5 ноутбуков с
  частично разной обвязкой.
- `tpe_study`: **собственная компактная реализация TPE** (`src/tpe_study/tpe.py`): ранговое
  разбиение по γ, 1-D KDE с адаптивной шириной + широкая prior-компонента, выбор кандидата по
  `log l(x) − log g(x)`. Все модификации — **флаги одной и той же реализации** ⇒ любая пара
  «`tpe` vs `tpe_*`» отличается ровно одним фактором (честный ablation).

---

## 3. Что сохранено без изменений (верность оригиналу)
- Функции и их аналитические градиенты (Sphere/Rosenbrock/Rastrigin/Ackley) — формулы 1-в-1.
- 4 формы веса и сама формула: `tanh`/`−tanh`/сигмоида·5/−сигмоида, `w=clip(1+0.2·z, 0.8, 1.2)`.
- Нормализация цели `f_norm=(f−f_min)/(f_max−f_min)`; шкалы raw/norm; данные clean/noisy_y.
- Гиперпараметры приведены к оригиналу: `n_init=10, gamma=0.2, n_candidates=24, max_evals=100`.

## 4. Что добавлено/исправлено по методологии
1. **Оценка качества всегда по raw clean функции.** Оптимизация может идти по raw/norm/noisy,
   но метрики (`dist_y`, `dist_x`, success) считаются по чистой raw — сравнение «яблоки к яблокам».
2. **Парный шум (common random numbers).** Шум зависит только от (функция, seed) и общий для всех
   методов и обеих шкал ⇒ корректные парные сравнения.
3. **Систематические стат-тесты.** Парный Уилкоксон каждой модификации vs `tpe` по seeds +
   поправка Холма; плюс робастность по 4 тестам × 3 поправкам × 2 метрикам. (В оригинале стат-тесты
   были только частично в fin_5; в fin_3/fin_4/fin_fin_* — отсутствовали.)
4. **Refinement сделан честным по бюджету.** В fin_5 `gradient_refine_steps=1` делал шаги спуска и
   **оценивал** предложенные точки (`self.evaluate(...)`) — это ТРАТИТ доп. вызовы objective.
   В `tpe_study` (`tpe_refine`, `tpe_gp_refine`) шаги делаются по оракулу ∇f **без** доп. оценок —
   ровно 1 оценка/trial, одинаковый бюджет со всеми методами.
5. **Классификация black-box / white-box.** Явно помечено, какие методы используют точный ∇f
   (`tpe_w_*`, `tpe_gp_w`, `tpe_refine`, `tpe_gp_refine`) и какие — нет (`random`, `tpe`, `optuna`,
   `tpe_gp`). refinement = по сути градиентный спуск с точным ∇f (верхняя граница, не улучшение TPE).
6. **Воспроизводимость.** `PYTHONHASHSEED=0`, фиксированные seeds, детерминированная оценка `f_max`,
   23 теста `pytest` (включая проверку, что инвариантность нормализации выполняется, и что refine
   снижает выпуклую цель). Seeds увеличены 30 → 50.
7. **Новые артефакты-таблицы:** `iteration_history.csv` (кривые по итерациям, mean/std),
   `raw_vs_norm_comparison.csv` (delta norm−raw, как `df_compare` в fin_3), `ablation_vs_tpe.csv`,
   `significance_*`, `all_results.xlsx`.

## 5. Что это меняет для выводов
- Результаты по весу w(x) в оригинале (fin_4/fin_5) опирались на **дефектный градиент** (x1=0).
  После исправления и при честных стат-тестах: **ни одна из 4 форм веса не даёт значимого улучшения**
  над baseline `tpe` (0/16, устойчиво к выбору теста).
- **GP-переранжирование** значимо улучшает baseline **без использования градиента** (6/16).
- **refinement** даёт больше всего значимых улучшений (12/16), но это ожидаемо: он эквивалентен
  градиентному спуску по точному ∇f.
- **Нормализация** инвариантна для `tpe`, всех форм веса и refine; масштабо-зависимы только GP-методы.
