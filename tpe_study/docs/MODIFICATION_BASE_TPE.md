# Моя модификация базового TPE: prior-компонента + адаптивная ширина ядра

Этот документ — самостоятельный раздел для курсовой по схеме
**входные данные → суть модификации → доказательства полезности**.
Все числа воспроизводимы: `PYTHONHASHSEED=0 python ablation_base.py`
(результат — `results/tables/ablation_base_tpe.csv` и `..._cells.csv`).

## 1. Входные данные (что оптимизируем и как меряем)

**Функции (2D, известный глобальный минимум):**

| Функция | bounds | x* | f* | σ шума (raw) | порог успеха по dist_y |
|---|---|---|---|---|---|
| Sphere | [−5,5]² | (0,0) | 0 | 0.005 | 1e−3 |
| Rosenbrock | [−2,2]×[−1,3] | (1,1) | 0 | 0.05 | 5e−2 |
| Rastrigin | [−5.12,5.12]² | (0,0) | 0 | 0.25 | 1.0 |
| Ackley | [−5,5]² | (0,0) | 0 | 0.05 | 0.5 |

**Факторы данных:** `scale` ∈ {raw, norm}, `data` ∈ {clean, noisy_y} (гауссов шум только в
наблюдаемом значении, общий для всех методов по (функция, seed) — парность).
**Метрика качества** считается ВСЕГДА по raw clean функции: `final_dist_y = |f(x_best) − f*|`,
где `x_best` — лучшая по НАБЛЮДАЕМОМУ значению точка (как выбрал бы пользователь).
**Бюджет:** `n_init=10, max_evals=100, gamma=0.2, n_candidates=24` (как в оригинальных ноутбуках).

## 2. Суть модификации (что именно я изменил в сэмплере TPE)

Базовый TPE строит по «хорошим» наблюдениям 1D-плотность `l(x)`, по «плохим» — `g(x)`,
и выбирает кандидата, максимизирующего `l(x)/g(x)`. Качество целиком зависит от того,
КАК оценена плотность. Я внёс в оценщик плотности (`WeightedKDE1D`, `src/tpe_study/tpe.py`)
два компонента:

1. **Широкая prior-компонента** (`use_prior`, по мотивам `consider_prior` Bergstra/Optuna).
   В смесь добавляется один широкий гауссиан в центре области с весом наравне с наблюдением.
   *Зачем:* без него «хорошая» плотность схлопывается в моду из нескольких близких точек,
   `l(x)` обнуляет хвосты, и TPE перестаёт исследовать — застревает в первой найденной яме.

2. **Адаптивная ширина ядра «magic clip»** (`adaptive_bandwidth`): ширина каждого ядра =
   расстоянию до ближайшего соседа в отсортированном ряду центров, с клипом `[1e−2·span, span]`.
   *Зачем:* в плотных (хороших) зонах ядра узкие → точный спуск к оптимуму; в разреженных —
   широкие → сохраняется исследование. Альтернатива (фиксированная ширина по Сильверману)
   либо размывает оптимум, либо не покрывает пустые зоны.

Оба компонента сделаны **ablat-able**: флаги `use_prior` и `adaptive_bandwidth` класса `TPE`
(дефолты `True` = мой вариант). Это позволяет честно отключать их по одному и доказывать вклад.

## 3. Доказательства полезности

### 3a. Ablation (отключаем компоненты, парный Уилкоксон + Холм, 20 seeds)
Полный вариант (`base_full`) vs три «увечья». Метрика — `final_dist_y` по raw clean,
8 ячеек = 4 функции × {clean, noisy_y}, scale=raw. «Значимо хуже» = p_holm<0.05 и медиана хуже.

| Отключено | Значимо хуже базы (из 8) | Медианное ухудшение |
|---|---|---|
| `fixed_bw` (нет адаптивной ширины) | **4 / 8** | +374 % |
| `naive_kde` (выключены ОБА) | **5 / 8** | +328 % |
| `no_prior` (нет prior) | 0 / 8 | +65 % |

Самые яркие ячейки (медиана final_dist_y, base → без компонента):
- **Sphere clean:** 0.029 → 0.91 без адаптивной ширины (×31, p_holm=2e−4).
- **Sphere noisy_y:** 0.014 → 0.96 без адаптивной ширины (×70, p_holm=1e−4).
- **Ackley noisy_y:** 0.67 → 3.86 при выключении обоих (p_holm=2e−3).

### 3b. Что показывает ablation (честная интерпретация)
- **Адаптивная ширина — главный драйвер точности** у оптимума: без неё точные функции
  (Sphere, Ackley) деградируют в десятки раз. Это значимо в 4/8 ячейках.
- **prior даёт стабильный, но по отдельности не значимый вклад** (медиана +65 % хуже без него,
  во всех 8 ячейках направление «хуже»); его роль — не дать плотности схлопнуться, и она
  яснее всего проявляется в комбинации (выключение ОБОИХ → 5/8 значимых ухудшений).
- **Нюанс честности:** на Rosenbrock фиксированная ширина оказалась чуть ЛУЧШE адаптивной
  (изогнутая долина), то есть адаптивность не универсально доминирует — но в среднем и на
  точных функциях выигрыш модификации значим.

### 3c. Юнит-тесты (механизм, детерминированно; `tests/test_tpe.py`)
- `test_prior_keeps_exploration_in_tail` — при узких ядрах prior держит плотность вдали от
  схлопнувшегося кластера (без него `logpdf` в хвосте ниже).
- `test_adaptive_bw_gives_narrow_kernel_in_dense_cluster` — в плотной зоне адаптивное ядро ýже
  фиксированного (Сильверман).
- `test_base_modification_beats_naive_kde_on_sphere` — полная база точнее «наивного» KDE на Sphere.
- `test_normalization_invariance_for_plain_tpe` — модификация не ломает инвариантность к
  нормализации цели (raw≡norm на clean).

## 4. Границы применимости (честно)
- Ablation проведена на raw-шкале, 2D, 20 seeds; для статьи можно расширить на norm и больше seeds.
- prior фиксирован (один широкий гауссиан, вес наравне с наблюдением); вес/ширина prior не варьировались.
- Вывод «адаптивная ширина важнее prior» получен на этих 4 функциях; на сильно анизотропных
  ландшафтах (Rosenbrock) картина мягче.
